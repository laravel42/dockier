# Dockier

> AI-native DevSecOps for modern engineering teams.

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![React](https://img.shields.io/badge/React-19-61DAFB?logo=react)](https://react.dev)
[![TypeScript](https://img.shields.io/badge/TypeScript-5.0-3178C6?logo=typescript)](https://www.typescriptlang.org)

**Secure. Analyze. Deploy.**

Dockier is an AI-powered DevSecOps platform that helps engineering teams secure their codebases, analyse architecture with AI, and ship faster — all from a unified dashboard.

- 🔐 **Security scanning** — Semgrep, SonarQube, and a custom rules engine across 10+ languages.
- 🤖 **AI analysis** — Auto-generated architecture docs, tech stack detection, and code-quality insights.
- 🧪 **Sensitive data detection** — Schema-aware PII, secret, and sensitive field classification.
- 📦 **Dependency intelligence** — Free CVE scanning via the OSV.dev database.
- 🚀 **Branch-based deployments** — Deploy to AWS and GCP with full history and rollback.

## Features

### AI project analysis
Auto-generated architecture documentation, tech stack detection, and code-quality insights — refreshed per commit.

### Security scanning
Semgrep, SonarQube, and a custom rules engine across 10+ languages with severity grouping.

### Sensitive data detection
Parses SQL, Prisma, Eloquent, TypeScript, and Python schemas to classify PII, sensitive, and secret fields.

### Dependency intelligence
Reads `package.json`, `composer.json`, `requirements.txt`, `Gemfile` and checks against the OSV.dev database.

### AI remediation
Generate fix merge requests directly from findings. Assign reviewers, ship without leaving the platform.

### Deployments
Branch-based deploys to AWS and GCP with full history, docker image tracking, and rollback.

## Getting started

### Prerequisites

- [Node.js](https://nodejs.org) 20+ (with `bun` recommended)
- A Lovable Cloud account (backend, auth, and storage)

### Installation

```bash
# Clone the repository
git clone https://github.com/dockier/dockier.git
cd dockier

# Install dependencies
bun install

# Configure environment variables
# The .env file is auto-generated by Lovable Cloud and should not be edited manually.
# Ensure VITE_SUPABASE_URL and VITE_SUPABASE_PUBLISHABLE_KEY are present.

# Start the development server
bun dev
```

The app will be available at `http://localhost:3000`.

### Build for production

```bash
bun run build
```

## Project structure

```text
src/
  components/          # Reusable UI components (shadcn/ui + custom)
  components/layout/   # Site header, footer, page shell
  components/marketing/# Landing page sections (hero, pricing, features, …)
  features/dashboard/  # Dashboard app under /app/*
    layout/            # App sidebar, topbar, shell
    components/        # Dashboard-specific components
    pages/             # Dashboard pages (overview, projects, security, …)
    fixtures/          # Mock data for UI development
    lib/types.ts       # Dashboard TypeScript types
  hooks/               # Shared React hooks
  integrations/        # Lovable Cloud, Supabase clients
  lib/                 # Utilities, site config, API helpers
  routes/              # TanStack Start file-based routes
  styles.css           # Global styles and design tokens
```
