# Contributing to Dockier

## Editorial conventions

These rules keep copy and metadata consistent across the marketing site.

### Page titles

- Format: `{Page} — Dockier` using an em-dash (`—`, U+2014), not a
  hyphen or en-dash.
- Use sentence case for the page name: `Product tour — Dockier`, not
  `Product Tour — Dockier`.
- Keep titles under 60 characters so they don't truncate in SERPs.

### Meta descriptions

- 150–160 characters. Unique per route — never reuse boilerplate.
- Lead with the value proposition; end with a clear noun phrase.

### Punctuation

- Em-dash (`—`) for parenthetical phrases and titles. No spaces around
  it in titles (`Pricing — Dockier`), spaces around it in body copy
  (`one platform — not five point tools`).
- Use straight quotes in code; curly quotes in prose are fine.

### Buttons and headings

- Sentence case (e.g. `Create new project`, not `Create New Project`).
- No trailing punctuation on button labels.

### Brand voice

- Friendly, professional, concrete. Avoid filler ("seamlessly",
  "leverage", "robust") unless it carries weight.
- Use realistic copy; never `Lorem ipsum`.
